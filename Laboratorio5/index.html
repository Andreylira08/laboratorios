¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Crea un recurso HEAD: Provee información general (metadatos) acerca del documento
POST: Crea un recurso PUT: Actualiza un recurso existente. No lo crea si no existe
PATCH: Aplica modificaciones parciales al recurso DELETE: Pregunta al servidor borrar un elemento en específico

Un error 404 significa que la página solicitada no se encontró en el servidor por lo que es trabajo del desarrollador corregir buscar el problema y resolverlo

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

El error 500 es una señal de que algo dentro del servidor no está funcionando correctamente como la base de datos de corrompio, un error de archivo, etc. Por lo que es el desarrollador debe buscar dentro del servidor para solucionar el problema

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

A diferencia de las versiones anteriores de HTML, que permitían crear sitios principalmente estáticos que necesitaban ser aderezados con CSS y JavaScript, HTML5 es mucho más dinámico e incluye elementos multimedia. Soporta de forma nativa el vídeo y el audio, e incluso puedes crear juegos o animaciones con él

¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

Es un sistema, automatizado o manual, que engloba a personas, máquinas y/o métodos organizados para recopilar, procesar, transmitir datos que representan información. En el cual engloba una serie de etapas en su ciclo de vida Planificación, análisis, Diseño, Implementación, Pruebas, Instalación, Uso y mantenimiento
Mientras que el ciclo de desarrollo es como un mapa que guía a través de cada etapa de desarrollo Fases: Planificación, análisis, diseño, desarrollo, pruebas, Implementación, mantenimiento.
